## Key Decisions Made

1. **Contracts in ntnx-common, not ntnx-promptforge** — The substrate (promptforge) implements contracts but doesn't own the definitions. ntnx-common is the single source of truth. Rust consumers validate against JSON Schema exports committed to their repo.

2. **Typed remediation enum, not command strings** — LLM free-text is parsed into a closed `RemediationAction` enum. Unparseable output always becomes `Escalate`. Enum variants produce explicit kubectl arg lists — never shell strings. Defense-in-depth against command injection.

3. **JSON Schema as cross-language contract** — Python Pydantic types are canonical. JSON Schema files are generated and committed to Rust repos. No code generation tooling — just schema conformance tests.

4. **Auth added to Rust HTTP server** — Deny-by-default matching ntnx-common's `DemoAuth` pattern. `NTNX_API_KEY` env var or `NTNX_AUTH_DEV_BYPASS=true`. Public routes (/health, /status, /metrics) remain unauthenticated.

5. **ntnx-codeforge now a proper Python package** — `pyproject.toml` with hatchling build, `ntnx-common` as dependency, `src/ntnx_codeforge/` layout. Bundle ingestion validates against `PromptBundle` contract before processing.
