## Key Decisions Made

1. **VSL3 not VSL4** — ioDrive2 uses iomemory-vsl (3.2.16), not iomemory-vsl4 (4.3.7). VSL4 rejects as "V6 hardware."
2. **Never NOP minimal mode flags** — causes kernel panics. Use `bypass_minimal_mode` ioctl-level approach instead.
3. **Pipeline pointer IS fio_device** — `nand_dev->0x1f0->deref` = same structure as fio_device. Offsets: +0x418=minimal_mode, +0x1288=PPS_store, +0x1e0=erase_blocks, +0x1d4=page_size, +0x11d0=mutex.
4. **PPS store structure (88 bytes):** `[0x00]=0x100000, [0x08]=0xfffffffff70000, [0x10]=erase_blocks, [0x20]=1, [0x40]=pool_ptr`
5. **PPS pool header (224 bytes):** `[0x00]=0x10000, [0x08]=list_head(needs init), [0xcc]=0x18, [0xd0]=page_data_size`
6. **Build on /data not /tmp** — /tmp is wiped on reboot, /data survives
7. **X1 Pro ready** — Ubuntu 24.04.4 + kernel 6.17, disable Secure Boot, r8125-dkms for ethernet, amdgpu.dcdebugmask=0x600 if boot hang
