## Key Decisions Made

1. **Auth is deny-by-default** — `NTNX_AUTH_DEV_BYPASS=true` required for anonymous access. No more silent fail-open.
2. **Bundle = subdirectory** — Each directory under `prompts/` is a PromptBundle. `policy.yaml` is optional per bundle.
3. **MCP uses stdio transport** — Most compatible with Claude Code and Cursor MCP (experimental).
4. **LLM in daemon loop** — The biggest architectural fix. Sentinel now actually diagnoses issues instead of creating empty-diagnosis tickets.
5. **Redis cooldowns** — SETEX with `cooldown_secs` TTL prevents duplicate ticket storms during recurring issues.
6. **Remediation gated by AutonomyMode** — DryRun logs, L1Auto executes pod restarts, L2Auto executes node drains. Observe/TicketOnly never execute.
7. **Templates stay hardcoded as fallback** — If promptforge is unreachable, sentinel falls back to inline prompts. No hard dependency.
