## Key Decisions Made

1. **`NTNX_AUTH_DEV_BYPASS=true` is the standard pattern** for all test files that exercise FastAPI apps using ntnx-common auth middleware. Applied to conftest.py or top of test files via `os.environ.setdefault()`.

2. **`from __future__ import annotations` is incompatible with pydantic-settings 2.13+ `bool` fields.** Removed from `ntnx-common/auth.py`. Python 3.12 supports `X | Y` union syntax natively, so `__future__` is unnecessary. Other repos checked — only ntnx-common was affected (other BaseSettings classes don't have `bool` fields).

3. **Silent exception pattern:** `except Exception as exc: # noqa: BLE001` + `LOG.debug("Suppressed: %s", exc)` before the `pass`/`continue`/`return`. This preserves the swallow-and-continue behavior while making failures visible in debug logs.

4. **costguard already has 82+ tests** (Rust unit + integration + shell hooks). The initial `python3 -m pytest` scan returned 0 because it's a Rust project — no new tests needed.

5. **ntnx-common installed in editable mode** (`pip install -e /opt/ntnx-common`) on orchestration-node. This means all repos importing `ntnx_common` get the live version. Intentional — the previous site-packages copy was stale and caused the bool field bug.
